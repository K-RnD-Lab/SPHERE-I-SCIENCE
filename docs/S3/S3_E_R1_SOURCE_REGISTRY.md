# S3-E-R1 Source Registry

## Research home

- Sphere: `S3`
- Lane: `S3-E Field Translation & Practical Deployment`
- Research line: `S3-E-R1 Field Translation Playbooks`

## Purpose

This file is the first evidence layer for `S3-E-R1`.

It tracks:

- which translation and adoption sources we actually use
- which sources are about advisory structure versus barriers
- which patterns are strong enough to shape field playbooks

## Current active scope

For the first execution pass, `S3-E-R1` is deliberately narrowed to:

- field translation of agricultural findings
- advisory formats and adoption logic
- barriers and enabling factors for practical deployment

This is a better first scope than trying to cover every extension system at once.

## Current research stance

At this stage, `S3-E-R1` should be treated as:

- a field-translation and deployment-readiness project

not as:

- a generic outreach guide
- a promise that adoption follows automatically from evidence

## What to collect next

The next useful sources should include:

- crop-specific advisory or extension case studies
- participatory field-monitoring programs
- farmer-facing toolkits with evidence-backed structure
- follow-up and checkpoint models after advisory deployment

## Claim discipline

From this registry, we may claim:

- that field translation depends on format, trust, adaptation, and follow-up
- that participatory and tailored advisory models are usually stronger than generic one-way delivery
- that barriers should be treated as design constraints, not afterthoughts

We should not yet claim:

- universal adoption rules
- that one playbook format works everywhere
- that translation quality alone guarantees field success
