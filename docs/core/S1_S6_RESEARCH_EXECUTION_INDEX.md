# S1-S6 Research Execution Index

## Purpose

This is the working index for running SPHERE I research step by step.

It answers three questions:

- what is already structurally fixed
- what is ready for real research execution
- what should stay mapped but not expanded yet

## Core rule

Do not try to capture every possible study in advance.

The current structure is enough when each sphere has:

- stable lanes
- enough research candidates
- at least one practical execution route
- clear evidence workflow

After that, the next step is not more planning. The next step is execution.

## What is not fixed yet

These are intentionally not fully fixed yet:

- every possible S1 therapeutic or oncology subcase
- full S6-B to S6-E execution packs
- every possible cross-sphere science plus technology case
- final public report locations for each study

Reason:

Expanding all of those now would create planning clutter. They should be opened only when a selected research line needs them.

## Step-by-step execution pattern

For each chosen study:

1. Open the five-file research pack.
2. Check `STRUCTURE_APPROVAL_GATE.md`.
3. Fill source registry with real sources.
4. Fill evidence schema with real rows.
5. Draft report sections from evidence only.
6. Add tables or figures.
7. Save active draft in `docs/S*/`.
8. Mirror stable report into the top-level `S` folder.
9. Move obsolete planning notes to `archived_docs/`.

## Decision

The current structure is sufficient to start real research execution.

The next useful task is not to add many more possible studies. The next useful task is to choose one route from the first practical wave and produce the first evidence-backed report.
