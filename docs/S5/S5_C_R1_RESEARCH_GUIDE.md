# S5-C-R1 Research Guide

## Research home

- Sphere: `S5`
- Lane: `S5-C Neurodegeneration Models`
- Research line: `S5-C-R1 Neurodegeneration State Models`

## Why this is a strong next `S5` study

`S5-C-R1` gives the sphere a disease-model layer after `S5-B` and `S5-E` establish vulnerability and time-based risk. It keeps neurodegeneration structured by states and biomarkers, not only disease names.

## Core question

How can neurodegenerative conditions be organized into state models using biomarker, imaging, pathology, and clinical-stage evidence?

## Practical substudies

### `S5-C-R1a` State Or Stage Comparison Logic

Goal:

- compare disease-stage frameworks across neurodegenerative conditions

Focus:

- AD biomarker staging
- Lewy body disease marker logic
- imaging and fluid-marker state definition

### `S5-C-R1b` Disease-Linked Molecular Or Biomarker Patterns

Goal:

- record disease-linked biomarker patterns with interpretation limits

Focus:

- amyloid, tau, neurodegeneration
- alpha-synuclein
- NfL, GFAP, p-tau markers
- protein-aggregate imaging

## Why this matters now

Neurodegeneration research is moving toward biomarker-based state definitions, blood tests, PET/imaging signatures, and disease staging. This makes `S5-C` a strong structural layer for research interpretation.

Useful references:

- biomarker-based staging of AD: https://www.nature.com/articles/s41582-024-00942-2

- PET imaging of protein aggregates: https://www.nature.com/articles/s41582-025-01126-2

## Data classes to collect

For the first useful pass, prioritize:

- disease or syndrome context
- biomarker class
- stage or state definition
- endpoint or clinical correlate
- assay or imaging modality
- interpretation limits

## Required outputs

The first useful output should include:

- neurodegeneration state-model evidence table
- biomarker class comparison panel
- stage-readiness note
- short disease-state interpretation report

## GitHub role

GitHub should hold:

- disease-state schema
- biomarker classification table
- stage comparison notes
- figures
- report

## Hugging Face role

Hugging Face should hold:

- neurodegeneration state explorer
- biomarker-stage comparison view
- disease-model interpretation dashboard

## Suggested first workflow

1. separate AD-specific staging from cross-disease neurodegeneration logic
2. extract biomarker class and modality
3. record whether the evidence defines pathology, stage, prognosis, or monitoring
4. flag assay and clinical-readiness limits
5. write a short disease-state model note

## Evidence fields to standardize

At minimum, track:

- `disease_context`
- `state_or_stage`
- `biomarker_class`
- `measurement_modality`
- `model_role`
- `evidence_source`
- `evidence_level`
- `interpretation_note`
- `limit_note`

## What not to do

Do not:

- reduce neurodegeneration to Alzheimer disease alone
- treat every biomarker as stage-defining
- ignore gray zones, cut-points, or assay differences
- mix diagnostic, prognostic, and monitoring roles without labels
