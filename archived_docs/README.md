# Archived Docs

This folder stores older or superseded documentation layers.

Typical examples:

- early operating models
- priority snapshots that were later expanded into lane maps
- interim summary maps
- structural notes that are no longer the live source of truth

These files remain useful for history and traceability, but they should not crowd `docs/`.

## Folder structure

`archived_docs/` is grouped the same way as `docs/`, but for superseded material:

- `archived_docs/core/`
  - cross-sphere historical strategy layers
  - older operating models
  - summary maps that are no longer live
- `archived_docs/S1/`
- `archived_docs/S2/`
- `archived_docs/S3/`
- `archived_docs/S4/`
- `archived_docs/S5/`
- `archived_docs/S6/`

## Archiving rule

Move a file from `docs/` to `archived_docs/` when:

- it has been replaced by a newer lane map or execution pack
- it no longer drives current research work
- it is still worth keeping for traceability or historical logic

Keep `archived_docs/` readable, but do not treat it as an active work surface.
